# ClearBag dev環境 手動E2Eテストチェックリスト

**作成日**: 2026-03-03
**対象URL**: https://clearbag-dev.firebaseapp.com/
**目的**: デプロイ後の基本動作を手動で確認するためのチェックリスト。今後テストケースを追加していく前提で運用する。

---

## 実行方法

1. **テスト前に環境をリセットする**（下記「セクション 0」参照）
2. 以下のチェックリストを上から順に実行する
3. 各項目の結果を `OK` / `NG` / `SKIP` で記録する
4. NG の場合は備考欄に再現手順・スクリーンショット等を添える
5. 新しいテストケースは該当セクションの末尾に追番で追加する

---

## 0. テスト前準備（環境リセット）

テスト実行前に必ず以下を実行し、Firestore と GCS を既知の状態に揃える。

```bash
PROJECT_ID=clearbag-dev GCS_BUCKET_NAME=clearbag-dev-clearbag-uploads-dev \
  uv run python scripts/reset_dev_data.py --email <your-dev-email>
```

**スクリプトが作成するシードデータ**:

| 種別 | 内容 |
|---|---|
| ファミリー | テストファミリー（plan: free） |
| プロフィール | 太郎（小学3年生、キーワード: サッカー,遠足） |
| ドキュメント1 | 運動会のおしらせ.pdf（完了） → イベント×1 + タスク×2 |
| ドキュメント2 | 保護者会のご案内.pdf（完了） → イベント×1 + タスク×1 |
| 合計 | ドキュメント2件、イベント2件、タスク3件（全て未完了） |

> **注意**: スクリプトは既存の families・users コレクションと GCS uploads/ を全削除する。dev 環境専用（clearbag-dev 以外では起動しない）。
